Tab代码补全¶

按下Tab键会在全局名字空间中搜索，并补全所输入的残缺变量字符串，譬如（输入my，然后按下Tab键）： 如果希望每输入一个字母实时给出代码提示，可以使用笔记本扩展插件nbextensions，后面会具体说明

查阅函数帮助¶

在Code Cell中输入?后跟函数名称（或其它任何对象名称），执行将弹出函数/对象的说明文档/详细信息，双问号??和?的区别在于，如果接的是函数，会给出函数的源代码，譬如：

import numpy as np
np.mat??

注，问号?在对象前或后都可以，习惯上我们在对象后面加上单/双问号。另外使光标停留在函数上，连续按一次或者连续按两次Shift+Tab可以弹出帮助

当前笔记本以PPT播放¶

可以使用jupyter内置的PPT选项，依次点击View-Cell Toolbar-幻灯片，之后每个单元格上面都会出现一个下拉选项，选Sub-Slide（将你希望作为幻灯片放映的Cell全部设为Sub-Slide，即子幻灯片），打开当前ipynb文件所在目录，Shift+右键打开cmd，执行：jupyter nbconvert <filename>.ipynb --to slides --post serve，将弹出幻灯片播放的网页，该方式会在ipynb文件所在目录下产生一个用于幻灯片的html文件，因此可以使用其它方式播放幻灯片
为了在播放的时候能够即时修改幻灯片，我们使用插件RISE，RISE用到reveal.js来运行PPT，可以在幻灯片模式下直接修改、运行Cell。下载RISE：conda install -c damianavila82 rise，然后安装插件并启用：

jupyter-nbextension install rise --py --sys-prefix
jupyter-nbextension enable rise --py --sys-prefix

重新打开ipynb文件，出现了一个新的插件按钮，点击即可播放PPT，很方便（仍要将需要参与播放的Cell设为Sub-Slide）：

提升性能的cython¶

安装：pip install cython，anaconda默认已经集成了该模块。使用Cython编译Python代码时，必须安装适当版本的C/C++编译器，Windows平台上就是Visual C++工具，要在jupyter的cell中使用cython，使用魔法%%cython（若报错Cell magic %%cython not found.请先执行：%reload_ext cython），但是出现错误：error: Unable to find vcvarsall.bat，这是因为我的机子上没有安装VC。查看适用于当前python版本的VC版本[7]： 根据图中所返回的14.0（内部版本号）找到对应的VS或VC版本： 由于我机子上正好安装了VS2015，因此我通过VS安装VC，安装的方式是，在控制面板-卸载，双击VS，点击“修改”，在编程语言中勾选C++最后安装即可。之后再使用EveryThing搜索关键字vcvarsall.bat就可以找到了
关于cython的使用请另见高级/性能分析/一些性能改进的措施.md

In [2]:
%reload_ext cython
In [3]:
#1,1,2,3,5,8,13,21,...
#返回不大于n的最大斐波那契数
def fib1(n):
    a,b=1,1
    while b<=n:
        a,b=b,a+b
    return a
In [4]:
%%cython
def fib2(int n):
    cdef int a=1
    cdef int b=1
    cdef int t
    while b<=n:
        t=a+b
        a=b
        b=t
    return a
In [5]:
%%cython
def fib3(n):
    a,b=1,1
    while b<=n:
        a,b=b,a+b
    return a

In [7]:

In [8]:
%timeit fib3(10)

上述示例中，即使在不对原python代码（fib1）做任何修改、仅仅添加一个%%cython的情况下（fib3），也能立即获得两倍的性能提升

自动打印全部变量值¶

默认情况下，在cell中最后一行如果是一个单独的变量，执行该cell会自动打印此变量值，你可能希望它能打印多个变量值且不限于最后一行，这是可以的：

In [9]:
from IPython.core.interactiveshell import InteractiveShell
InteractiveShell.ast_node_interactivity = 'all' #要恢复旧貌，赋值'last'，具体见[14]
In [14]:
x='Hello'
y='DaiYang'
x
y
Out[14]:
'Hello'
Out[14]:
'DaiYang'

如果你想要所有的jupyter实例（Notebook或Console）都设置该选项，只需创建~/.ipython/profile_default/ipython_config.py文件并写入一下内容（其中~为用户文件夹，Windows为C:\Users\Administrator\）：

c = get_config()
# Run all nodes interactively
c.InteractiveShell.ast_node_interactivity = "all"

扩展插件nbextensions¶

安装插件包集合：conda install jupyter_contrib_nbextensions
这些扩展大多是以JavaScript编写的，加载到你的前端笔记本前端界面上，以增强笔记本的功能。要启用这些插件，使用子命令：

jupyter nbextension enable <nbextension require path>

插件包在目录anaconda\pkgs\jupyter_contrib_nbextensions-0.5.1-py36_0\Lib\site-packages\jupyter_contrib_nbextensions\nbextensions\下（使用everything搜索关键字jupyter_contrib_nbextensions即可找到），每个插件文件夹下都有一个特别的js文件，以codefolding为例，在该文件夹下有一个main.js（要么是main.js，要么是和插件名同名的js文件），于是codefolding的nbextension require path就是：codefolding/main，同理，插件hinterland文件夹下有hinterland.js，其nbextension require path就是：hinterland/hinterland.js，各个插件文件夹下都有自述文件（readme.md），描述其具体功能。现在我们启用codefolding：

C:\Windows\System32>jupyter nbextension enable codefolding/main
Enabling notebook extension codefolding/main...
      - Validating: ok

另外禁用的子命令为disable，具体请执行jupyter nbextension --help查阅
为了避免上述繁琐的操作，安装插件管理工具：conda install jupyter_nbextensions_configurator
启动笔记本，页面出现该管理界面： 常用插件（请在管理面板自行查阅相关插件使用说明）：

Freeze¶

将Cell设为只读（Read-Only）或者冰冻（Freeze）
对Code Cell，只读：代码可以执行但是无法修改；冰冻：代码不可以执行和修改
对MarkDown Cell，只读：markdown源码可以通过双击查阅，但是不能修改；冰冻：无法查阅markdown文本的源码，不能修改
通常将Code Cell置为Read-only，将MarkDown Cell置为冰冻
譬如我将1.3标题置为只读，1.4标题置为冰冻，上面的“小埋鼠”代码示例置为只读，“肥宅快乐水”代码示例置为冰冻
